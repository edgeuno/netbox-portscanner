Scanner Runtime

Port scanning, stripped down to the supported path.

This page is now an operator overview for the supported scanner path. Configuration values, endpoints, and credentials are intentionally kept out of the UI.

Supported Backend
VMPortScannerQueue
Single supported runtime entrypoint for the cleaned plugin.
Execution Path
The supported control flow from cron or shell into the scanner worker.
1. Runner
Use the repository wrapper when scheduling execution.
2. Management Command
The only supported command target is `portscanner`.
3. Backend
`VMPortScannerQueue` performs the live scan and service sync.
Run Command
Tenant name stays explicit so scheduled jobs are easy to audit.
/opt/netbox/venv/bin/python /opt/netbox/netbox/manage.py portscanner <tenant>
`portscanner_runner.sh` is the preferred repository wrapper for cron execution.
Operator Notes
The home page is intentionally neutral and does not expose deployment-specific settings.
No config echo
Endpoints, users, tokens, passwords, and SSL flags are not rendered here.
No legacy registry UI
Removed Proxbox pages and model-backed flows are not part of this screen.
Runtime first
Use this page for orientation, then validate plugin loading and command execution in NetBox.
Validation Focus

Plugin load
Confirm NetBox starts with the plugin enabled and the page renders.
Command discovery
Confirm `manage.py portscanner <tenant>` is available in the target environment.
Compile sanity
Keep `python -m compileall netbox_portscanner` as the minimum repo-side verification.
